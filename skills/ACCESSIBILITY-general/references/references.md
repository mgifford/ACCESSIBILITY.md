## Official Standards

- **WCAG Guidelines** — <https://www.w3.org/WAI/standards-guidelines/wcag/>  
  Official Web Content Accessibility Guidelines — the authoritative reference for all accessibility requirements.  
  License: W3C Software License

- **W3C Web Accessibility Initiative (WAI)** — <https://w3.org/WAI/>  
  The W3C group that develops WCAG, ATAG, UAAG, and ARIA; the home of all accessibility standards.  
  License: W3C Software License

- **WAI-ARIA Authoring Practices Guide (APG)** — <https://www.w3.org/WAI/ARIA/apg/>  
  Canonical patterns, roles, states, properties, and keyboard interaction guidance for accessible UI components.  
  License: W3C Software License

- **W3C WCAG Understanding Docs** — <https://www.w3.org/WAI/WCAG22/Understanding/>  
  Official explanatory guidance for each WCAG 2.2 success criterion: intent, benefits, examples, and related techniques.  
  License: W3C Software License

- **W3C WCAG Techniques** — <https://www.w3.org/WAI/WCAG22/Techniques/>  
  Official sufficient/advisory techniques and common failure patterns mapped to WCAG success criteria.  
  License: W3C Software License

## Community Resources

- **The A11Y Project** — <https://www.a11yproject.com/>  
  Community-driven accessibility resource with guides, checklists, and articles for practitioners.

- **WebAIM** — <https://webaim.org>  
  Non-profit offering accessibility tools, training, evaluation services, and the widely-cited annual Screen Reader User Survey.

- **MDN Web Docs** — <https://developer.mozilla.org/>  
  Definitive resource for web standards including comprehensive ARIA and accessibility API documentation.
